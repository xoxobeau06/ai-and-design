The Call Stack Resolves

In Magic, spells resolve in reverse order—last in, first out. This is identical to the JavaScript Call Stack.

When you invoke a function, it gets pushed onto the Stack. If that function calls another, the new function is pushed on top. Nothing underneath can resolve until the topmost spell leaves the stack.

When an error occurs, you trace the stack trace downward—just as you trace a complicated counter-war on the battlefield.

console.log("Draw Step"); 
// Sync: Blocks main thread

Promise.resolve().then(() => {
    console.log("Resolve Ability");
}); 
// Async: Goes to Micro Queue

Sorcery Speed: Sync Execution

The parser reads line by line. The main thread blocks. No other plays can be made until the execution phase completes.

Instant Speed: Async Microtasks

Promises and callbacks are sent to the Event Loop queues. The main thread continues, and these resolve immediately when the stack is empty, jumping ahead of macrotasks.

The Stack (Event Loop)

The Event Loop is the referee, checking if the stack is clear to allow the next queue resolution to fire onto the battlefield.
